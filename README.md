# Криптографические технологии

Репозиторий с решенными заданиями по предмету "Криптографические технологии" с использованием Java. 

## Описание проекта
Проект включает в себя три задания, каждое из которых демонстрирует различные аспекты криптографических алгоритмов и технологий. Задания охватывают реализацию шифрования сообщений с помощью шифра Цезаря, алгоритма RSA для криптографии с открытым ключом и генерацию безопасных паролей.

## Структура репозитория
Ссылки на решения каждого задания и их условия представлены ниже:

### Первое задание
#### Условие
Реализовать методы для шифрования шифром Цезаря и шифром усложненной перестановки для чисел в диапазоне от 20 до 50.

- [Первое задание](src/task1.java)

### Второе задание
#### Условие
1. Реализовать программу для шифрования и дешифрования сообщений с помощью алгоритма RSA:
   1.1 Простые числа p и q задаются пользователем.
   1.2 Проверка, являются ли p и q простыми числами.
   1.3 Число e, проверяющее взаимную простоту с Ф(n).
   1.4 Число d вычисляется алгоритмом Евклида.
   1.5 Каждая буква в сообщении представляется числом его позиции в алфавите.
2. Проверить правильность работы программы, сравнив результаты шифрования.

- [Второе задание](src/RSA.java)

### Третье задание
#### Условие
Данная программа предназначена для оценки стойкости паролей в информационных системах, вычисления минимальной длины пароля и генерации безопасного пароля. Используя заданные параметры, программа рассчитывает количество возможных паролей и генерирует случайный пароль, который соответствует требованиям безопасности.
**Вводные данные:**

- P (Вероятность подбора пароля): Вероятность того, что злоумышленник сможет подобрать пароль. В данной задаче задано значение 10e-6.
- V (Скорость подбора паролей): Количество паролей, которые злоумышленник может проверять в минуту. В данной задаче задано значение 10 паролей в минуту, что эквивалентно 10.0 / 60 паролей в минуту.
- T (Максимальный срок действия пароля): Период, в течение которого пароль считается действительным. В данной задаче установлен на 5 дней, что эквивалентно 5 * 24 * 60 минутам.
- A (Мощность алфавита): Количество различных символов, используемых для формирования пароля. В данной задаче задано значение 62 (26 строчных букв, 26 заглавных букв и 10 цифр).

**Программа выполняет следующие действия:**

- Выводит значения всех переменных и их пояснения.
- Вычисляет минимально допустимое количество возможных паролей (S*).
- Определяет минимальную длину пароля (L), соответствующую S*.
- Генерирует случайный пароль заданной длины.
- Сохраняет сгенерированный пароль в текстовый файл, предварительно проверяя наличие файла и удаляя его, если он существует.

- [Третье задание](src/PasswordUtility.java)

2. Перейдите в директорию проекта: `cd kgt`
3. Выберите нужный день: `cd src/task1.java`
4. Запустите нужный файл Java.
